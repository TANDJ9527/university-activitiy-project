# 校园活动发布平台 - 需求文档

## 1. 项目概述

### 1.1 项目背景
校园活动发布平台是一个专为校园环境设计的活动管理与发布系统，旨在为学生、教师和校方组织提供便捷的活动发布、浏览和管理的平台。该平台支持多种活动类型的发布，包括讲座、文体活动、志愿服务、社团活动、竞赛等，帮助校园社区更好地组织和参与各类活动。

### 1.2 项目目标
- 提供用户友好的活动发布和管理界面
- 支持多维度搜索和筛选功能
- 实现安全的用户认证和权限管理
- 确保数据安全性和系统稳定性
- 支持响应式设计，适配多种设备

### 1.3 主要用户群体
- **学生用户**：可以浏览活动、发布个人活动、编辑自己的活动
- **校方用户**：可以发布官方活动、管理活动信息
- **平台管理员**：可以管理所有活动和用户

## 2. 功能需求

### 2.1 用户认证与管理
- **用户注册**：支持邮箱注册，选择角色（学生/校方），设置昵称/组织名称（最大100字），密码至少6位
- **用户登录**：邮箱+密码登录，JWT令牌认证
- **用户资料管理**：查看和更新个人信息
- **权限控制**：基于角色的访问控制

### 2.2 活动管理
- **活动发布**：
  - 必填字段：标题（最大120字）、活动说明（最大8000字）、开始时间
  - 可选字段：地点、主办方、联系方式、结束时间、活动类别
  - 自动记录发布者信息和发布时间
- **活动编辑**：发布者可以修改自己的活动信息，平台管理员可以修改任何活动
- **活动删除**：发布者可以删除自己的活动，平台管理员可以删除任何活动
- **活动浏览**：分页显示活动列表，支持排序

### 2.3 活动搜索与筛选
- **关键词搜索**：支持在标题、描述、地点、主办方中搜索
- **类别筛选**：按活动类型筛选（讲座、文体、志愿服务、社团、竞赛、其他）
- **发布者筛选**：按发布来源筛选（学生/校方/全部）
- **排序功能**：按发布时间、开始时间排序

### 2.4 活动报名和取消
- **活动报名**：用户可以报名参加活动，系统会记录报名信息
- **活动取消**：用户可以取消报名，系统会更新报名状态为已取消

### 2.5 活动评论
- **活动评论**：用户可以在活动详情页评论活动，系统会记录评论信息
- **评论管理**：平台管理员可以管理所有评论，包括删除和审核

### 2.6 活动收藏
- **活动收藏**：用户可以在活动详情页收藏活动，系统会记录收藏信息
- **收藏管理**：平台管理员可以管理所有收藏，包括删除和审核收藏

## 4. 用户角色与权限

- 搜索和筛选活动
- 发布个人活动
- 编辑/删除自己的活动

## 5. 数据模型

### 5.1 用户表 (users)
- id: CHAR(36) (主键)
- email: VARCHAR(255) (唯一)
- password_hash: VARCHAR(255)
- display_name: VARCHAR(100)
- role: ENUM('student', 'school')
- created_at: DATETIME(3) (默认 CURRENT_TIMESTAMP(3))

### 5.2 活动表 (activities)
- id: CHAR(36) (主键)
- user_id: CHAR(36) (外键)
- publisher_role: ENUM('student', 'school')
- title: VARCHAR(120)
- description: TEXT
- location: VARCHAR(500) (默认空字符串)
- organizer: VARCHAR(200) (默认空字符串)
- contact: VARCHAR(200) (默认空字符串)
- category: VARCHAR(50) (默认'其他')
- start_at: DATETIME(3)
- end_at: DATETIME(3) (可选)
- created_at: DATETIME(3) (默认 CURRENT_TIMESTAMP(3))
- updated_at: DATETIME(3) (默认 CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3))

### 5.3 平台管理员表 (platform_admins)
- user_id: CHAR(36) (主键，外键)
- note: VARCHAR(200) (可选)
- created_at: DATETIME(3) (默认 CURRENT_TIMESTAMP(3))

## 6. API 接口规格

### 6.1 认证接口
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息

### 6.2 活动接口
- `GET /api/activities` - 获取活动列表（支持查询参数：q=关键词, category=类别, sort=排序方式, publisher=发布者类型）
- `GET /api/activities/:id` - 获取单个活动详情
- `POST /api/activities` - 创建新活动
- `PUT /api/activities/:id` - 更新活动
- `DELETE /api/activities/:id` - 删除活动

### 6.3 系统接口
- `GET /api/health` - 健康检查
